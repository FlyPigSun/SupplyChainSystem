# 备份管理

> 一句话：数据库文件的复制备份与恢复，管理员可操作，恢复前自动备份当前数据。

## 核心功能

- 备份列表查看（文件名、大小、创建时间）
- 手动创建备份（复制 .db 文件到 backups/ 目录）
- 恢复备份（恢复前自动创建当前数据的保护备份）
- 安全校验（防止路径遍历攻击）

## 涉及文件

| 层级 | 文件 |
|------|------|
| 后端路由 | `backend/src/routes/backups.js` |
| 前端视图 | `frontend/src/views/Backups.vue` |
| 数据文件 | `backend/data/supply_chain.db` |
| 备份目录 | `backend/backups/` |
| 定时脚本 | `backend/scripts/backup.sh` |

## 核心逻辑

### 创建备份
```javascript
const timestamp = new Date().toISOString().replace(/[:.]/g, '-');
const backupName = `backup_${timestamp}.db`;
fs.copyFile(DB_PATH, path.join(BACKUP_DIR, backupName));
```

### 恢复备份
1. 安全校验：文件名必须符合 `/^[a-zA-Z0-9_\-:.]+\.db$/`，防止路径遍历
2. 先备份当前数据：`before_restore_${timestamp}.db`
3. 复制备份文件覆盖 `supply_chain.db`
4. 关闭旧数据库连接，重新初始化（使后续请求使用恢复后的数据库）

### 定时备份
`backend/scripts/backup.sh` 通过系统定时任务（crontab/launchd）每天执行：
- 保留最近 30 天的备份
- 自动清理过期文件

## 数据流

```
管理员操作 ──► backups.js
                  │
                  ├──► 创建 ──► fs.copyFile ──► backups/backup_xxx.db
                  │
                  └──► 恢复 ──► 1. 备份当前 → before_restore_xxx.db
                                2. 复制备份 → supply_chain.db
                                3. 关闭旧连接 → getDatabase() 重新初始化
```

## 关键设计决策

- 备份即文件复制，不通过 SQL dump，保持 SQLite 的完整性
- 恢复前强制创建保护备份，防止误操作导致数据丢失
- 恢复后关闭旧连接并重新初始化，避免进程继续使用旧数据库句柄
- 文件名正则校验杜绝路径遍历攻击
- 定时备份与手动备份使用同一目录，统一管理
