# 账号管理

> 一句话：基于 JWT 的认证系统，支持用户 CRUD、角色管理、密码修改、账号停用/启用。

## 核心功能

- 登录（JWT Token，有效期 30 天）
- 修改密码（需旧密码验证）
- 用户列表查询
- 新增用户（默认密码 123456，首次登录强制修改）
- 停用/启用用户
- 修改用户角色（admin / user）
- 重置密码（重置为 123456）

## 核心逻辑

### 认证流程
```
1. POST /api/auth/login { username, password }
2. bcrypt.compareSync() 校验密码
3. 检查用户状态（disabled → 拒绝登录）
4. jwt.sign({ id, username, role }, JWT_SECRET, { expiresIn: '30d' })
5. 返回 token + userInfo（含 must_change_pwd）
```

### 权限中间件
- `authMiddleware`：校验 JWT Token，提取用户信息挂载到 `req.user`
- `adminMiddleware`：校验 `req.user.role === 'admin'`，否则返回 403

### 角色与权限

| 角色 | 权限 |
|------|------|
| admin | 全部功能（产品创建/修改/删除、价格创建/修改、用户管理、备份恢复） |
| user | 查看类操作（产品列表、价格查询、用量计算、成本核查、日志查看） |

### 密码安全
- 密码使用 bcrypt 哈希存储（salt rounds = 10）
- `must_change_pwd` 字段：1 表示首次登录必须修改密码，修改后设为 0
- 默认密码：`123456`

### 用户管理约束
- 不能停用自己
- 不能停用 `admin` 账号
- 不能修改 `admin` 的角色
- 不能修改自己的角色

## 数据流

```
Login.vue ──► auth.js (login)
                  │
                  ▼
           JWT Token + must_change_pwd
                  │
                  ▼
           authStore (Pinia) 存储 token + 用户信息
                  │
                  ▼
           Layout.vue ──► 检查 must_change_pwd ──► 强制修改密码弹窗
                  │
                  ▼
           后续请求携带 Authorization: Bearer <token>
                  │
                  ▼
           authMiddleware 校验 token ──► 挂载 req.user
```

## 关键设计决策

- JWT 有效期 30 天，减少频繁登录；Token 遗失风险通过账号停用控制
- `must_change_pwd` 机制保证新创建/重置密码的用户必须立即修改
- 权限控制精确到路由级别，admin 专属路由统一使用 `adminMiddleware`
- 用户状态与角色分离：`status` 控制账号是否可用，`role` 控制功能权限
- 安全约束在服务端强制执行（不能停用自己等），不依赖前端校验
