# 配料表管理

> 一句话：产品配料的层级解析与展示，支持 Excel 导入/导出、搜索高亮、按产品类型/生产工厂筛选。

## 核心功能

- 配料列表（分页、关键词搜索、按产品类型/生产工厂筛选、排序）
- 配料详情（层级树形展示）
- 配料层级自动解析（一级/二级）
- Excel 导入/导出
- 模板下载

## 核心逻辑

### 配料解析规则（parseIngredient）

核心判定：括号内是否含分隔符（顿号/逗号）。含分隔符 → 拆分为二级配料；否则整体作为一级配料。

| 配料示例 | 解析结果 | 原因 |
|----------|----------|------|
| `复配水分保持剂（三聚磷酸钠、焦磷酸钠）` | 一级 + 二级 | 括号内含顿号分隔 |
| `黄油（≥2.5%）` | 一级（整体） | 括号内无分隔符，含量标注 |
| `海苔酥脆松（鸡肉）` | 一级（整体） | 括号内无分隔符，口味说明 |

实现细节：
1. 找到第一个开括号，配对闭括号（处理嵌套）
2. 判断括号内容是否为含量标注（正则：`/^(添加量)?[≥＞><≤≈≥]?\d+(\.\d+)?[%％克\/份]*$/`）
3. 判断括号内是否含顿号/逗号分隔符
4. 按 `splitRespectingParens()` 拆分（跳过括号内分隔符）
5. 合并不应该拆分的项（如 `"单"`、`"双"` + 后续含 `甘油/酸/酯` 的项）

### 导入流程
1. 按「物品编码」分组读取 Excel
2. 逐个调用 `parseIngredient()` 解析配料
3. 同一产品内按 `(ingredient, level)` 去重
4. 删除 Excel 涵盖的产品已有数据，重新插入

### 列表展示
前端将数据库中的扁平配料记录构建为层级字符串：
- 一级配料加粗 `<strong>`
- 有二级配料的一级配料显示为 `一级（二级1、二级2）`
- 含括号的二级配料：名称加粗，括号内容不加粗

## 数据流

```
Excel 导入 ──► parseIngredient() ──► product_labels (扁平存储)
                                           │
ProductLabels.vue ◄── 查询 + buildIngredientStr() ◄──┘
```

## 关键设计决策

- 数据库扁平存储（每条配料一行，带 `level` 和 `parent_ingredient`），不存树结构
- 前端动态构建层级字符串，灵活支持多种展示需求
- 导入时全量替换（先删后插），保证数据与 Excel 完全一致
- `splitRespectingParens()` 使用深度计数器处理嵌套括号，避免误拆分
