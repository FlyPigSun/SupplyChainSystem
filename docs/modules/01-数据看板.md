# 数据看板

> 一句话：纯前端统计页面，聚合展示产品、原料、价格、供应商等核心数据概览及 TOP10 排行。

## 核心功能

- 统计卡片：产品总数、原料种类、价格记录
- TOP10 排行：原料使用频率、供应商覆盖产品数、品牌使用频次
- 全部数据来自前端 API 聚合调用，无独立后端路由

## 涉及文件

| 层级 | 文件 |
|------|------|
| 前端视图 | `frontend/src/views/Dashboard.vue` |
| 依赖 API | `products` / `prices` 路由的已有接口 |

## 核心逻辑

页面加载时并行调用：
1. `GET /api/products` 获取全部产品及其配方原料
2. `GET /api/prices` 获取全部价格记录

在前端内存中统计：
- **原料使用 TOP10**：遍历所有产品的 `materials`，按 `material_name` 聚合，统计每个原料出现在多少个产品中
- **供应商覆盖 TOP10**：遍历所有产品的 `materials`，按 `supplier` 聚合
- **品牌使用 TOP10**：遍历价格库，按 `brand` 聚合

## 数据流

```
Dashboard.vue ──► 调用 productsApi.getAll() + pricesApi.getAll()
                        │
                        ▼
               前端内存统计聚合
                        │
                        ▼
               渲染统计卡片 + 三张 TOP10 表格
```

## 关键设计决策

- 无独立后端统计接口，减少后端复杂度，前端内存聚合足够（当前数据量千级）
- 响应式布局：`el-col :xs="12" :sm="8"` 适配移动端
