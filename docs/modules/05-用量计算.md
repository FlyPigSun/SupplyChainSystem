# 用量计算

> 一句话：根据产品配方和数量计算原料总用量，自动匹配价格并计算成本，支持人工修正记忆。

## 核心功能

- 多产品用量计算（选择产品 + 输入数量）
- 三维度汇总展示（按产品、按原料、按供应商）
- 价格自动匹配（4 级优先级）
- 匹配修正记忆（人工修正自动生效）
- Excel 模板导入（批量导入产品-数量清单）

## 核心逻辑

### 计算流程
```
1. 接收 [{ productId, quantity }]
2. 加载全部价格库 allPrices
3. 加载全部修正记录 corrections → 构建 correctionMap（key: name||brand_spec||supplier）
4. 遍历每个计算项：
   a. 查询产品配方 materials
   b. matchPrices(materials, allPrices) → 4 级匹配
   c. 检查 correctionMap，如有修正则替换为修正后的价格
   d. calcWeightInPriceUnit(totalWeight, pricePerKg, priceUnit, spec) → 单位换算
   e. cost = weightInPriceUnit * pricePerKg
5. 返回所有原料行数据
```

### 价格匹配（priceMatcher.js）
4 级优先级匹配（详见 [07-匹配修正](./07-匹配修正.md)）：
1. `exact`：brand + supplier 完全一致
2. `brand_model`：brand + model 匹配
3. `supplier_model`：supplier + unified_name/model 匹配
4. `fuzzy`：仅型号匹配，品牌不同

### 修正记忆
修正记录的 key 为 `{material_name}||{brand_spec}||{supplier}`，匹配时优先查询：
```javascript
const correctionKey = `${m.material_name}||${m.brand_spec || ''}||${m.supplier || ''}`;
const correction = correctionMap[correctionKey];
```
如命中修正，matchType 变为 `corrected`，cost 使用修正后的价格计算。

### 单位换算
通过 `unitConversion.calcWeightInPriceUnit()` 处理：
- g → kg：除以 1000
- kg → g：乘以 1000
- 箱/袋/桶等单位：从 `spec` 中提取重量，换算为 kg

### 模板导入
模板列：序号、产品名称、供应商、数量
- 产品名称必须存在于数据库
- 供应商必须属于该产品配方中的供应商
- 按 `产品+供应商` 合并数量

## 数据流

```
用户输入 / Excel导入 ──► calculator.js ──► matchPrices() ──► unitConversion
                                                    │              │
                    material_prices ◄───────────────┘              │
                    match_corrections ◄────────────────────────────┤
                                                                   ▼
                                                            返回 cost + matchType
                                                                   │
                                                            Calculator.vue
```

## 关键设计决策

- 计算时一次性加载全部价格和修正记录，避免循环查库（N+1 问题）
- 修正记录按 `(source_name, source_brand_spec, source_supplier)` 唯一索引，INSERT OR REPLACE 更新
- 计算结果中保留 `originalMatch` 字段，展示修正前的原始匹配结果
- 模板导入严格校验产品名称和供应商的精确匹配，错误信息包含数据库中的可用值建议
