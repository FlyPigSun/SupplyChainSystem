# 产品配方

> 一句话：产品的核心数据结构管理，支持 CRUD、配方原料管理、Excel 批量导入/导出。

## 核心功能

- 产品列表查询（支持按关键词、类型、销售状态筛选）
- 产品 CRUD（编码、名称、类型、规格、销售状态、生产工厂）
- 配方原料管理（12 个字段：原料名称、统一名称、品牌规格、品牌、供应商、生产商、产地、执行标准、对应比例、单个克重、单位）
- Excel 配方导入（14 列模板，支持 upsert/skip 模式）
- Excel 配方导出（按产品分组，每行一个原料）
- 产品类型列表、生产工厂列表（从配料表管理去重获取）

## 核心逻辑

### 产品列表查询
```sql
SELECT * FROM products WHERE (name LIKE ? OR code LIKE ?) AND type = ? AND sales_status = ?
ORDER BY updated_at DESC
```
查询结果再 JOIN `product_materials` 按 `product_id` 分组组装。

### 产品创建/更新
- 创建：`INSERT INTO products` → `insertMaterials()` 批量插入配方原料
- 更新：`UPDATE products` → `DELETE FROM product_materials WHERE product_id = ?` → 重新 `insertMaterials()`
- 配方更新策略：**全量替换**（先删后插），保证数据一致性

### 配方导入（recipeImport.js）
1. 按「物料编码」分组读取 Excel 行
2. `mode = 'upsert'`：已存在则更新产品信息 + 替换配方；不存在则新建
3. `mode = 'skip'`：已存在跳过
4. 字段映射：物料编码 → code，产品名称 → name，产品类型 → type，规格 → unit，原料名称 → material_name，统一名称 → unified_name，品牌规格 → brand_spec，品牌 → brand，供应商 → supplier，原料生产商 → manufacturer，产地 → origin，执行标准 → standard，对应比例 → ratio，单个克重(g) → weight

### 配方导出
- 按产品分组，每个原料占一行
- 14 列输出：物料编码、产品名称、产品类型、规格、供应商、原料名称、统一名称、品牌规格、品牌、原料生产商、产地、执行标准、对应比例、单个克重(g)

## 数据流

```
Excel 导入 ──► recipeImport.js ──► products + product_materials
                                           │
Products.vue ◄── products.js ◄─────────────┘
     │
     ▼
Excel 导出
```

## 关键设计决策

- 产品编码 `code` 为 UNIQUE，导入和创建时校验重复
- 配方原料更新采用全量替换策略，避免增量合并的复杂性和数据不一致
- 产品与销售状态解耦：`sales_status` 字段支持 `on_sale` / `off_sale`
- `factory_name` 从 `product_labels.supplier` 去重获取，保证与配料表数据一致
