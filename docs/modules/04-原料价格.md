# 原料价格

> 一句话：原料价格的维护中心，支持 CRUD、Excel 批量导入，是价格匹配引擎的数据源。

## 核心功能

- 价格列表查询（支持关键词模糊搜索：品牌、型号、供应商、原料类型）
- 价格创建/更新（按 `brand + model + supplier` 唯一键，存在则更新，不存在则新建）
- 价格删除
- Excel 批量导入（8 列模板）

## 核心逻辑

### 价格唯一键
以 `(brand, model, supplier)` 为唯一标识：
```sql
SELECT id FROM material_prices WHERE brand = ? AND model = ? AND supplier = ?
```
- 存在 → UPDATE（更新 category/spec/unit/price/remark）
- 不存在 → INSERT（自动生成 material_name = `{brand} {model} {supplier}`）

### 导入字段映射
| Excel 列名 | 数据库字段 | 说明 |
|------------|-----------|------|
| 原料类型 | category | 分类 |
| 品牌 | brand | 品牌名称 |
| 型号 | model | 产品型号 |
| 供应商 | supplier | 供应商名称 |
| 规格 | spec | 规格描述 |
| 单位 | unit | 默认 kg |
| 最小规格单价 | price | 必填，必须 > 0 |
| 备注 | remark | 备注信息 |

### 价格匹配数据源
`material_prices` 是 `priceMatcher.js` 的唯一直接数据来源，包含字段：
- `id`, `material_name`, `category`, `brand`, `model`, `supplier`, `spec`, `unit`, `price`, `remark`

## 关键设计决策

- `brand + model + supplier` 三元组作为唯一键，覆盖同一品牌不同型号、同一型号不同供应商的场景
- 导入时支持增量更新（已存在则覆盖），不删除已有记录
- `material_name` 自动拼接生成，作为展示用名称
- 价格为必填且必须 > 0，空价格或 0 价格在匹配时视为无效
