# 成本核查

> 一句话：上传 BOM 成本核算 Excel，自动比对系统配方、匹配价格、检测成本占比是否在合理区间。

## 核心功能

- BOM 成本核算表上传解析
- 模板表头结构校验（4 个区域：产品组成、包材组成、单个成品组成、BOM 成本合计）
- 模板数据非空/合法性校验（数值范围、金额公式校验、百分比范围）
- 价格差异比对（Excel 含税单价 vs 系统价格库）
- 成本占比预警（动态配置规则，支持占比区间 + 金额绝对值区间）
- 编码乱码检测与自动修复
- 匹配修正记忆支持

## 核心逻辑

### 解析流程
```
1. 上传 Excel → XLSX 解析为 rows（header:1 模式，原始行列数组）
2. validateTemplateHeaders() → 4 区域表头结构校验
3. validateTemplateData() → 非空 + 合法性校验
4. parseAuditSheet() → 提取原料数据 + 成本组成 + 产品信息
5. 产品名称匹配（精确匹配 → 模糊匹配，支持编码修复）
6. 价格匹配 + 修正记忆
7. 成本占比预警
9. 返回完整结果
```

### 4 个区域的数据提取
| 区域 | 列范围 | 提取内容 |
|------|--------|----------|
| 产品组成 | C~I 列 | 原料名称、品牌/型号、重量(g)、含税单价、不含税单价、金额、百分比 |
| 单个产品包材组成 | A~I 列 | 包材名称、数量、含税单价... |
| 单个成品组成 | A~I 列 | 工艺分项、组成、不含税单价、金额、百分比 |
| BOM 成本合计 | A~I 列 | 项目、金额、百分比 |

### 金额公式校验
校验逻辑：`金额 ≈ 重量(g) × 含税单价(元/kg) / 1000`
- 绝对误差 ≤ 0.05 或 相对误差 ≤ 5% 视为通过
- 金额为 0 时跳过（允许免费样品）

### 价格差异比对
使用 `matchPricesForAudit()`（成本核查专用匹配），额外检测：
- `flavor_diff`：品牌匹配但口味不同（如奶酪味 vs 原味）
- 单位换算：通过 `unitConversion.convertToStandardUnit()` 将系统价格统一为 元/kg 后比较
- 差异阈值：绝对差值 > 0.5 视为有差异

### 成本占比预警
规则配置在 `cost-rules.json`，每次请求时重新读取：
```json
{
  "rules": [
    { "id": "food_cost", "label": "食材成本", "keywords": ["食材", "原料"], "minPercent": 45, "maxPercent": 60 },
    { "id": "labor", "label": "人工费用", "keywords": ["人工"], "minPercent": 10, "maxPercent": 20 },
    ...
  ]
}
```
匹配方式：按 `keywords` 在 `costBreakdown` 中模糊匹配费用项名称。

## 数据流

```
Excel 上传 ──► bomCheck.js
                  │
                  ├──► validateTemplateHeaders() ──► 结构校验
                  ├──► validateTemplateData() ────► 数据校验
                  ├──► parseAuditSheet() ─────────► 提取原料/成本
                  ├──► 产品名称匹配 ──────────────► products
                  ├──► 价格匹配 ─────────────────► priceMatcher + material_prices
                  ├──► 修正记忆查询 ─────────────► match_corrections
                  ├──► 成本占比预警 ─────────────► cost-rules.json
                  │
                  ▼
            返回完整核查结果
```

## 关键设计决策

- BOM 模板结构校验优先于数据校验，结构不对直接返回错误列表
- 金额公式校验使用双重容错（绝对误差 + 相对误差），避免小数精度问题
- 产品名称支持编码乱码修复（Latin-1 → UTF-8 回退），文件名作为兜底
- 成本预警规则热加载（每次请求重新读文件），修改后无需重启服务
- 分节识别同时支持 A列工艺大类（胚体/成型/冷加工）和 B列动态组成名（老面/酥皮/馅料等）
