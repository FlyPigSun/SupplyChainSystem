# 操作日志

> 一句话：记录系统中关键操作（产品 CRUD、导入导出、价格更新、成本核查等），支持查询和前端主动记录。

## 核心功能

- 操作日志列表查询（支持关键词搜索、数量限制）
- 前端主动记录操作日志（用于纯前端计算类操作）

## 涉及文件

| 层级 | 文件 |
|------|------|
| 后端路由 | `backend/src/routes/logs.js` |
| 工具模块 | `backend/src/utils/logOperation.js` |
| 前端视图 | `frontend/src/views/Logs.vue` |
| 数据表 | `operation_logs` |

## 核心逻辑

### 日志记录方式

有两种记录途径：

1. **后端路由内直接调用**（推荐）：
   ```javascript
   const { logOperation } = require('../utils/logOperation');
   await logOperation(req.user.username, 'create_product', `创建产品: ${name}`);
   ```

2. **前端主动 POST**：
   ```javascript
   POST /api/logs { action: 'calculate', detail: '计算 5 个产品用量' }
   ```

### logOperation 工具
封装了统一的 INSERT 操作，内置错误静默处理（日志失败不影响主流程）：
```javascript
async function logOperation(operator, action, detail) {
  try {
    await runAsync(
      'INSERT INTO operation_logs (operator, action, detail) VALUES (?, ?, ?)',
      [operator, action, detail || '']
    );
  } catch (err) {
    // 静默处理，不抛错
    console.error('记录操作日志失败:', err.message);
  }
}
```

### 已记录的操作类型

| action | 触发场景 |
|--------|----------|
| `create_product` | 创建产品 |
| `update_product` | 更新产品 |
| `delete_product` | 删除产品 |
| `export_products` | 导出产品配方 |
| `add_price` | 新增价格 |
| `update_price` | 更新价格 |
| `import_prices` | 导入原料价格 |
| `import_recipes` | 导入配方 |
| `import_product_labels` | 导入配料表 |
| `export_product_labels` | 导出配料表 |
| `bom_check` | 成本核查 |
| `calculate` | 用量计算（前端记录） |
| `change_role` | 修改用户角色 |
| `create_backup` | 创建备份 |
| `restore_backup` | 恢复备份 |

## 数据流

```
各业务路由 ──► logOperation() ──► operation_logs
                    │
         Logs.vue ◄─┘
```

## 关键设计决策

- 日志记录失败不阻断主业务流程（静默处理）
- 前端主动记录用于纯计算类操作（如用量计算），后端记录用于数据变更操作
- 日志表无复杂查询需求，简单时间倒序 + 关键词搜索即可
- 日志量较大时可通过 `LIMIT` 控制返回数量，默认 100 条
